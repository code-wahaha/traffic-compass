# 流量罗盘 · 桌面客户端打包配方

## 一键打包

```powershell
cd "F:\Vibe Coding\抖音监察\webapp"
pyinstaller --noconfirm --clean --windowed --name "流量罗盘" `
  --add-data "static;static" --add-data "rules;rules" --add-data "audit_skill.md;." `
  --collect-submodules uvicorn `
  --collect-all webview --collect-all pythonnet --collect-all clr_loader main.py
```

> ⚠️ `--collect-all webview pythonnet clr_loader` 不能省：pywebview 的 Windows 窗口引擎（.NET 桥接/WebView2 DLL）是动态加载的，
> PyInstaller 扫不出来。少了它们 exe 会"双击没反应"（服务起了但窗口崩、进程静默退出）。
> ⚠️ 带链接提取需再加 `--collect-all playwright`（提取依赖本机已装的 ms-playwright 浏览器和抖音登录态）。
> ⚠️ **不要**打包 dashscope SDK——其加密依赖(cryptography/pyo3)在中文路径的打包环境里必崩(Utf8Error panic)，
> extract.py 已改为直接调百炼 REST 接口，无此依赖。
> ⚠️ main.py 已内置 `PLAYWRIGHT_BROWSERS_PATH=%LOCALAPPDATA%\ms-playwright`——打包后的 playwright
> 默认会到包内部找浏览器(.local-browsers)，必须指回系统目录。产物目前在 `dist_new\流量罗盘\`。

## 重打包三条铁律（踩过的坑）

1. **打包前必须确认 exe 进程全死透**（`Get-Process 流量罗盘 | Stop-Process -Force` 后等 2 秒）——旧进程占着 DLL 会让 PyInstaller 删旧目录删一半失败，产出"僵尸包"（_internal 根目录 DLL 全空，双击报 Failed to load Python DLL）。
2. **别用管道吞退出码**——`pyinstaller … | tail` 会把失败掩盖成功；要单独 `echo $?` 验证。
3. **打完必查** `_internal\python313.dll` 存在才算成功。
4. windowed 模式 stdout/stderr 是空句柄：main.py 里已重定向到 `_data\app.log`（playwright 起子进程也依赖这个修复），排障先看这个文件。

产物：`dist\流量罗盘\`（整个文件夹就是软件，压 zip 发人即可）
入口：`dist\流量罗盘\流量罗盘.exe`

## 架构

`main.py` = 客户端入口：后台线程起 FastAPI（自动找空闲端口）→ pywebview 开原生窗口（WebView2 内核，Win10/11 自带）。界面就是 static/index.html，用户看不到浏览器痕迹。

## 路径规则（config.py 里定的）

- 开发模式：数据在 `webapp\_data\`
- 打包模式：数据在 **exe 旁边的 `_data\`**（compass.db + apikeys.json），用户拷走 _data 就是全量备份
- static/rules/audit_skill.md 打进包内（只读资源）

## ⚠️ 发包前铁律

1. **绝不带你自己的 key**：检查 `dist\流量罗盘\` 下没有 `_data\` 文件夹（打包不会带，但手工测试后可能生成——发人前删掉！）。新用户首次打开会看到"填 DeepSeek key"引导卡（BYOK）。
2. 链接提取在客户端包里自动禁用（DISABLE_EXTRACT=1，playwright 不打包，体积原因），用户粘文案用。
3. 想换名字/图标（白标）：`--name "别的名字" --icon xxx.ico`。

## 测试要点

双击 exe → 原生窗口弹出（无地址栏）→ 填 key 卡（新机器）→ 自查 → 锁预测 → 发布后表格 → 导出 CSV。
