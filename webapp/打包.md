# 流量罗盘 · 桌面客户端打包配方

## 发行版三步(2026-07-07 起的标准流程)

```powershell
cd "F:\Vibe Coding\抖音监察\webapp"

# ① 打 exe(产物在 dist_release\流量罗盘\;不碰 dist_new——那是作者自用、可能正开着)
pyinstaller --noconfirm --clean --windowed --name "流量罗盘" `
  --distpath dist_release --workpath build_release `
  --add-data "static;static" --add-data "rules;rules" `
  --add-data "audit_skill.md;." --add-data "xiaohongshu_skill.md;." `
  --collect-submodules uvicorn `
  --collect-all webview --collect-all pythonnet --collect-all clr_loader `
  --collect-all playwright main.py

# ② 把浏览器塞进包里(客户机没装 playwright 浏览器,不塞=提取/扫码登录全废)
#    版本必须对上 playwright 的 browsers.json 钉的修订号(现在=1223):
$lb = "dist_release\流量罗盘\_internal\playwright\driver\package\.local-browsers"
mkdir $lb -Force
Copy-Item "$env:LOCALAPPDATA\ms-playwright\chromium-1223" "$lb\chromium-1223" -Recurse
Copy-Item "$env:LOCALAPPDATA\ms-playwright\chromium_headless_shell-1223" "$lb\chromium_headless_shell-1223" -Recurse
Copy-Item "$env:LOCALAPPDATA\ms-playwright\winldd-1007" "$lb\winldd-1007" -Recurse
# (headless_shell 给提取用,完整 chromium 给"扫码登录"弹窗用,两个都要;ffmpeg 不用带)

# ③ MVP 实测 → 清隐私 → 压包(脚本自动剔 _data/钥匙/抖音登录/账本/日志并自检)
#    测试会在 dist_release\流量罗盘\ 下生成 _data,压包前删干净!
python make_release_zip.py    # 产物 = 桌面\流量罗盘安装包.zip(含使用说明.txt)
```

> main.py 的浏览器定位逻辑(2026-07-07 改):**包内 .local-browsers 优先**(发行版),
> 没有再指回系统 `%LOCALAPPDATA%\ms-playwright`(作者机),都没有就降级提示。
> 所以同一份代码,作者机不塞浏览器也能跑,客户机塞了浏览器就全功能。

## 重打包铁律(踩过的坑,一条都不能少)

1. **打包前确认目标 dist 目录的 exe 进程死透**(`Get-Process 流量罗盘 | Stop-Process -Force` 后等 2 秒)——旧进程占 DLL 会产出"僵尸包"(_internal 根目录 DLL 全空,双击报 Failed to load Python DLL)。发行流程用独立的 dist_release,天然避开正在跑的 dist_new。
2. **别用管道吞退出码**——`pyinstaller … | tail` 会把失败掩盖成成功;单独 `echo $?` 验证。
3. **打完必查** `_internal\python313.dll` 存在才算成功。
4. `--collect-all webview pythonnet clr_loader` 不能省:少了=“双击没反应”。
5. **不要打包 dashscope**——cryptography 在中文路径打包环境必崩;extract.py 已改百炼 REST 直调。
6. windowed 模式 stdout/stderr 是空句柄:main.py 已重定向到 `_data\app.log`,排障先看它。
7. **发包前隐私三查**(make_release_zip.py 会自动做,但要看它报"干净"):
   无 `_data\`(apikeys/compass.db/app.log/chrome_data)、无 `douyin_cookies.json`、无 records。

## 架构

`main.py` = 客户端入口:后台线程起 FastAPI(自动找空闲端口)→ pywebview 开原生窗口(WebView2 内核,Win10/11 自带)。界面就是 static/index.html。

## 路径规则(config.py 里定的)

- 开发模式:数据在 `webapp\_data\`
- 打包模式:数据在 **exe 旁边的 `_data\`**(compass.db + apikeys.json + chrome_data),用户拷走 _data 就是全量备份
- static/rules/audit_skill.md/xiaohongshu_skill.md 打进包内(只读资源)

## 测试要点(发包前 MVP)

双击 exe → 原生窗口弹出 → 三样引导卡(新机器)→ 粘文案审核出报告 → 右下角「抖音」按钮状态正确 → 锁预测 → 发布后表格 → 导出 CSV → **删掉测试产生的 _data 再压包**。
